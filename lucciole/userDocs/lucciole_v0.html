Lucciole Sprint1

Introduction

Realizzazione in qak di una simulazione delle lucciole

Requirements

  • Il sistema deve disporre di una serie di lucciole
    • 

  • Le lucciole devono essere attori indipendenti, realizzati con il liguaggio qak
    • 

  • Le lucciole emettono un flash a intervalli costanti (uguale per tutte)
    • 

  • La fase iniziale è randomica
    • 

  • In maniera autonoma, ogni lucciola deve osservare i flash delle altre per poter sincronizzarsi, 
affinchè tutte le lucciole emettano luce allo stesso istante
    •

    Requirement analysis

    A seguito dell'analisi, è stato deciso di implementare la lucciole come QActors, con 3 stati:

QActor lucciola context ctxlucciolelocale {
    State s0 initial {
    	// Nello stato iniziale si inizializza 
        // in maniera randomica la fase della lucciola.
    }
    Transition t0
        whenTimeVar Phase -> flash

     State flash {
        // Nello stato di flash, la lucciola emette un flash
        // e scatena un evento per notificare le altre.
	    println("$name | Flash") color yellow
	    emit flash : flash()
	}
	Transition t0
	    whenTimeVar Phase -> flash
	    whenEvent flash -> onFlash
	
	State onFlash {
        // Nello stato di onFlash, la lucciola osserva i flash delle altre
        // e si sincronizza di conseguenza.
	    onMsg(flash : flash(Time)){
	    }
	}
	Transition t0
	    whenTimeVar Phase -> flash
	    whenEvent flash -> onFlash 
}

    Problem analysis

    A seguito dell'analisi del problema è stato deciso di implementare l'interazione tra le lucciole in
un contesto locale per motivi di velocità di sviluppo.
Inoltre l'interazione tra le lucciole è deciso tramite un evento apposito.


Event flash : flash(Time)

Context ctxlucciolelocale ip [host="localhost" port=8010]
